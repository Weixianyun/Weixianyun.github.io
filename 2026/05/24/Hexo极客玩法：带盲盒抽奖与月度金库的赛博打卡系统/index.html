谁说静态博客只能看文章？在折腾完 Keep 极简主题后，我决定更进一步，为我的 [Weixianyun’s Blog] 引入一套完整的“赛博打卡”生态。

这次的升级不仅包含之前的打卡记录，还引入了 Canvas 刮刮乐盲盒、商业级加权概率抽奖 以及 月度收益金库 功。

1. 核心功能拆解

为了保持 Hexo 的纯净，这套系统完全基于浏览器前端逻辑：

  • 打卡系统： 利用 LocalStorage 记录连续足迹。
  • 刮刮乐盲盒： 使用 HTML5 Canvas 的 destination-out 混合模式实现涂层擦除。
  • 权重抽奖算法： 并非简单的随机，而是设定了 49% 谢谢参与、25% 十元、3% 五十元等精确概率。
  • 赛博金库： 自动累加每月中奖金额，并具备“跨月自动清零结算”的逻辑判断。

2. 商业级抽奖概率逻辑

在代码中，我放弃了平均随机，改用“区间划分法”来实现真实的掉落感：

function drawPrize() {
  const rand = Math.random() * 100; // 掷出一个 0-100 的点数
  if (rand < 49) return "🤣 谢谢参与";     // 49% 概率
  if (rand < 74) return "💰 10元";         // 25% 概率
  if (rand < 89) return "💴 20元";         // 15% 概率
  if (rand < 97) return "💵 30元";         // 8% 概率
  return "💎 50元大奖！";                  // 3% 锦鲤概率
}
3. 月度金库与响应式布局
为了让“金库”在右侧美观显示，我使用了 Flexbox 弹性布局，使其在电脑端横向并排，而在手机端自动垂直堆叠。

同时，通过记录 wallet_month 标签，系统会在你每个月第一次打开页面时，自动判断是否需要开启新一轮的收集周期。
