Hexo常用命令清单

这是我在搭建个人博客过程中整理的 Hexo 与 Git 常用运维命令。把它记录在这里，作为以后日常写作和网站维护的“备忘录”。

🛠️ 场景一：日常写作与本地预览（最高频）

每次打开电脑准备写博客，第一件事永远是进入博客文件夹 (cd my-blog)，然后再执行其他操作。

  • hexo new "你的文章标题"
    • 用途： 新建文章。Hexo 会自动在 source/_posts 目录下生成一个带有当前时间和基础配置的 .md 草稿文件。
  • hexo server (简写：hexo s)
    • 用途： 启动本地微型服务器。在浏览器输入 http://localhost:4000 即可实时预览正在写的文章或修改的配置。

🚀 场景二：发布上线与源码备份（双管齐下）

当在本地预览满意，准备向全世界公开时，需要打出这套完整的发布“连招”。

1. 发布网页成品（更新外网博客）

hexo clean     # 大扫除：清除本地之前生成的旧网页缓存
hexo generate  # 网页生成：把 Markdown 文章打包编译成静态 HTML 文件
hexo deploy    # 一键发射：把生成的静态网页推送到 GitHub Pages
